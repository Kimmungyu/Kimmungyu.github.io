With the Embold platform, not only can customer discover critical hotspots in their software systems, it can be a platform to drive innovation and improve software quality and transparency within research and development division.
Embold looks at the code from multiple dimensions:

  • Design: Design issues are patterns that seem to work but are counter-productive and far from optimal in practice. Design issues can easily result in un-maintainable and error-prone solutions. Embold identifies multiple class and method level anti-patterns in the code which takes dependencies, inheritance hierarchy and important software metrics into consideration. Few Examples: God class, Brain methods, Local breakable, Local butterfly, Shotgun surgery.
  • Metrics: Metrics are a way to control quality. A Software Metric is a measure of some property of a piece of software or its specifications. It compresses system traits into numbers
  • Duplication: Duplicate Code or a Clone is a piece of software code that occurs more than once, either within a same component (which could be a single class, source code file, a module) or across multiple components of an application or even across multiple applications owned or maintained by the same entity. Code Duplication refers to not just exact matching between two identical pieces of code, but also to the matching functionality, which can be possibly extracted out into a separate component.
  • Code Issues: Code issues are programming flaws like inefficient coding practices, sub-optimal resource utilization, improper exception handling, etc. These flaws affect KPIs like robustness, accuracy, efficiency, maintainability, reliability, understandability etc.

Embold uses a proprietary ‘multi-vector’ diagnostic technology to combine those different dimensions to analyse and prioritize hotspots in software components. This helps you fix the biggest issues first.

After introducing Embold as core development platform, clients have observed a significant improvement in their overall software quality regarding several different metrics and KPIs. In addition, the number of raised issues by their customers have significantly decreased, leading to reduced costs and a higher customer’s satisfaction. As a side effect, after making the Embold rating a part of the developer’s regular performance evaluations, their skillset has noticeably increased leading to a higher development quality.
